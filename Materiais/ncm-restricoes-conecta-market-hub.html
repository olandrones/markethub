O Problema

Por que NCM e restrições importam

Sem o código NCM correto, todo o estudo de viabilidade é inválido. Sem análise de restrições, o cliente pode selecionar produtos que não podem ser importados.

01

Classificação NCM

O código de 8 dígitos do Sistema Harmonizado determina alíquotas de importação (II, IPI), exigências regulatórias (anuências) e medidas de defesa comercial. A tabela possui ~10.000 a 15.000 códigos ativos.

02

Restrições Regulatórias

Produtos podem exigir licenças de 10+ órgãos (ANVISA, IBAMA, INMETRO, ANATEL...) ou estar sujeitos a anti-dumping — 49 medidas ativas contra a China. O cliente precisa saber disso antes de selecionar.

03

Risco de Erro

Classificação incorreta gera multa de 1% do valor aduaneiro (mín. R$ 500, máx. 10% da DI). A Receita Federal já usa IA (SISAM) para fiscalizar — automação sem validação humana é risco inaceitável.

Posição no Pipeline

Esses módulos entram entre o Step 1b (enriquecimento do catálogo) e o Step 3 (seleção de produtos), evitando que produtos não-importáveis avancem no funil. A classificação acontece antes da pesquisa de preços na China.

Arquitetura do Pipeline

4 etapas de classificação e análise

Pipeline híbrido: IA econômica para triagem, IA multimodal para casos ambíguos, APIs oficiais para restrições e IA para consolidação de alertas.

Etapa 1 — Pré-classificação NCM

LLM econômico analisa descrição e características em batch (5-10 por chamada), retornando top 3-5 NCMs com score de confiança e justificativa técnica.

Tier 3: GPT-4.1-nano / GPT-4o-mini ~$0.001/produto Batch 5-10 produtos

Se confiança ≥ 90%

Pula direto para a Etapa 3 (lookup de restrições). O modelo Tier 3 resolve ~80% dos casos.

  • Capítulo NCM inequívoco
  • Descrição detalhada com material e uso

Se confiança < 90%

Produto segue para a Etapa 2 — refinamento com análise visual da foto. ~20% dos casos.

  • Material aparente ambíguo
  • Múltiplas posições NCM possíveis

Etapa 3 — Lookup de Restrições

Etapa 100% programática — sem LLM. Consulta APIs e bases locais para identificar anuências, anti-dumping e alíquotas.

$0.00 LLM Classif API + CSV MDIC + IBPT IBPT = interim (aproximado)
JavaScript — Lookup com Fallback
async function checkRestrictions(confirmedNcm, country = 'CN') {

  // 3A — Tratamento administrativo (anuências)
  // Classif API — requer certificado digital ICP-Brasil
  const anuencias = await classifAPI
    .getTratamentoAdministrativo(confirmedNcm);

  // 3B — Anti-dumping (tabela local, sync semanal do CSV MDIC)
  const antidumping = await antidumpingDB
    .query({ ncm: confirmedNcm, country });

  // 3C — Alíquotas
  // ⚠️ IBPT = fonte INTERIM (aproximada, metodologia própria)
  // ✅ Classif API = fonte OFICIAL (Receita Federal)
  const taxes = certificadoDisponivel
    ? await classifAPI.getAliquotas(confirmedNcm)
    : await ibptAPI.getAliquotas(confirmedNcm, uf);
  // ⚠️ Diferenças de 2-3% possíveis entre IBPT e oficial

  const severity = calculateSeverity(anuencias, antidumping);
  return { ncm: confirmedNcm, anuencias, antidumping, taxes, severity };
}

Anti-Dumping: NCM + País de Origem

Regra Fundamental

Direitos anti-dumping são aplicados por NCM + país de origem — não por fornecedor individual. Se o NCM X da China tem anti-dumping de 34%, aplica-se a qualquer fabricante chinês desse produto.

IBPT: Fonte Interim vs. Classif: Fonte Oficial

A IBPT API fornece alíquotas aproximadas (metodologia própria do instituto) — é fonte interim até o certificado digital estar disponível. A Classif API (Portal Único Siscomex) é a fonte oficial da Receita Federal. Diferenças de 2-3% possíveis entre IBPT e dados oficiais — pode impactar a viabilidade de produtos no estudo DDP.

Pré-requisitos de Infraestrutura

Antes do R2: Certificado digital e-CNPJ A1 (~R$ 200/ano) Antes do R2: Registro CNPJ na IBPT (gratuito) R2: Job de sincronização CSV anti-dumping MDIC
Modelo de Dados

Novas entidades e fontes

Três entidades adicionadas ao modelo do Conecta Market Hub para suportar classificação NCM e restrições.

01

NcmClassification

Relacionamento 1:1 com CatalogProduct. Armazena NCMs sugeridos pela IA (top 3-5 com scores), NCM confirmado pelo operador, método de classificação (Tier 3, Tier 1 multimodal, manual), confiança da IA e feedback de correção para melhoria contínua.

02

ImportRestriction

Relacionamento 1:N com CatalogProduct. Cada restrição inclui tipo (anuência, anti-dumping, proibição), órgão responsável, necessidade de LPCO, alíquota anti-dumping (se aplicável), severidade (livre/licença/bloqueado), validade e fonte oficial.

03

Tabelas de Referência

NcmTable (~15.000 códigos), AntidumpingMeasure (143 medidas vigentes) e TratamentoAdministrativo sincronizados com APIs e CSVs públicos. NCM anual, anti-dumping semanal, tratamento administrativo semanal, alíquotas conforme fonte (IBPT semestral / Classif contínua).
